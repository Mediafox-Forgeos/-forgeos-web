## ADR-0001–0004 status

**Explicitly confirmed still true as of this document's generation:** ADR-0001 ("ForgeOS evolves with products"), ADR-0002 ("EV Platform is White Label"), ADR-0003 ("Kylum is Pilot Customer"), and ADR-0004 ("Technology Stack") exist **only** as UI fixture objects in `apps/forgeos-web/data/decisions.ts` — a hardcoded array feeding the ForgeOS `/decisions` screen. No corresponding file exists under `docs/adr/`. `docs/adr/README.md` itself notes this: "ADR-0001 through ADR-0004 are currently recorded as fixture data in the ForgeOS workspace... Formal ADR documents will be migrated to this directory as part of Mission 003" — and Mission 003 is the same mission that was skipped (see Group C, [Product Atlas](./MOVOS_PRODUCT_ATLAS_v1.0.md)). This document does **not** promote them to formal ADRs; that migration remains undone and is out of scope here.

## The surviving original plan

The four-phase roadmap recorded as UI fixture data on day one — **Foundation → Core Platform → OCPP → Pilot** (`apps/forgeos-web/data/roadmap.ts`) — has not been rewritten since it was first committed. It still accurately describes exactly where the product sits today: Foundation is done, Core Platform (the EV domain) hasn't started, OCPP and Pilot are further out. The original plan was not wrong; it simply has not been executed past its first phase yet.

For the full term-by-term classification against the five domain-modeling categories (Implemented / Documented / Historical / Fixture-only / Candidate) required for M001-A, see [`docs/domain/M001-A_UBIQUITOUS_LANGUAGE_v0.1.md`](../domain/M001-A_UBIQUITOUS_LANGUAGE_v0.1.md) — this document recovers the history; that one formalizes it for domain modeling.
